模块规范

Node.js模块规范遵循CommonJs模块规范。

我们新建3个js文件index1.js，index2.js负责导出 test1.js, test2.js负责导入。

exports.add导出的是一个对象，故应该以对象的形式导入。 另外不可以exports后直接赋值因为require导入的永远都是module.exports导出的，默认是空对象。exports是module.exports的引用，如果赋值就改变了指向，require导入的就只是一个空对象了

rquire细节

导入格式：require（X）

  • 情况一x是一个核心模块

    • 直接返回核心模块，并且停止查找

  • 情况二x是以./或../

    1. 将x当作一个文件在对应的目录下查找
      • 如果有后缀名，按照后缀名的格式查找对应的文件
      • 如果没有后缀名，会按照如下顺序
        1. 直接查找文件X
        2. 直接查找x.js
        3. 查找X.json
        4. 查找X.node
    2. 没有找到对应的文件的话，将x作为一个目录
      • 查找目录下的index文件
        1. 查找X/index.js
        2. X/index.json
        3. X/index.node

情况三名称并不是路径，也不是模块，这时会去node_modules中查找，然后自动去找index文件 如果找不到node_moudles,会去上层目录去找，知道找到根目录

npm世实际上就是将源码下载到node_moudles中

nodejs事件

Node.js 所有的异步 I/O 操作在完成时都会发送一个事件。

所有这些产生事件的对象都是 events.EventEmitter 的实例，实例化生成的对象叫做事件触发器。events是内置模块，EventEmitter是events里面的一个类

buffer缓冲区

Buffer 是内存区域，一个专门存放二进制数据的缓冲区。

当我们从一端读取一个文件，并将它转化为二进制，另一端吸收速度不如读取速度，此时需要一个缓冲区

stream流

流是为 Node.js 应用程序提供动力的基本概念之一。

流是一种以高效的方式处理读/写文件、网络通信、或任何类型的端到端的信息交换。

stream （模块）提供了构建所有流 API 的基础。 所有的流都是EventEmitter的实例。

如图所示，文件一部分先转化为二进制存入到缓冲区，只要缓冲区有数据，就可以直接使用，大大节约了时间

显然，可读流更快

使用stream模块创建可读流

Stream 有四种流类型：

  • Readable - 可读操作。
  • Writable - 可写操作。
  • Duplex - 可读可写操作。
  • Transform - 操作被写入数据，然后读出结果。

流常用的事件有：

  • data - 当有数据可读时触发。
  • end - 没有更多的数据可读时触发。
  • error - 在接收和写入过程中发生错误时触发。
  • finish - 所有数据已被写入到底层系统时触发

可写流可以将数据写入文件

管道流

管道提供了一个数据从输出流到输入流的机制。

我们使用管道可以从一个流中获取数据并将数据传递到另外一个流中。

举例：复制文件

我们把文件比作装水的桶，而水就是文件里的内容，我们用一根管子(pipe)连接两个桶使得水从一个桶流入另一个桶，这样就慢慢的实现了大文件的复制过程。

管道流可以链式调用，但需满足pipe后返回的流时可读的

fs模块

fs 模块提供了许多非常实用的函数来访问文件系统并与文件系统进行交互。

文件系统（fs 模块）模块中的方法均有异步和同步版本，例如读取文件内容的函数有异步的 fs.readFile() 和同步的 fs.readFileSync()。

异步读取

同步读取

同步读取会阻塞程序进行，推荐异步读取

fs打开文件 参数：文件路径，打开方式，文件权限，回调函数

fs.open(path, flags[, mode], callback)

Flag 描述
r 以读取模式打开文件。如果文件不存在抛出异常。
r+ 以读写模式打开文件。如果文件不存在抛出异常。
rs 以同步的方式读取文件。
rs+ 以同步的方式读取和写入文件。
w 以写入模式打开文件，如果文件不存在则创建。
wx 类似 ‘w’，但是如果文件路径存在，则文件写入失败。
w+ 以读写模式打开文件，如果文件不存在则创建。
wx+ 类似 ‘w+’， 但是如果文件路径存在，则文件读写失败。
a 以追加模式打开文件，如果文件不存在则创建。
ax 类似 ‘a’， 但是如果文件路径存在，则文件追加失败。
a+ 以读写追加模式打开文件，如果文件不存在则创建。
ax+ 类似 ‘a+’， 但是如果文件路径存在，则文件读取追加失败。

r，w，默认在文件开头出进行操作，a默认在文件末尾进行操作。

path模块

path模块用于对路径和文件进行处理，提供了很多好用的方法

path.resolve规则

path.resolve()方法会将一个或多个路径片段的序列解析为一个绝对路径，给定的路径的序列是从右往左被处理的，后面每个path被依次解析，直到构造完成以恶绝对路径；如果处理完所有给定的path段后还没有生成句对路径，则使用当前工作目录；生成的路径被规范化并删除尾部斜杠，零长度path段被忽略；如果没有path传递段，path.resolve（）将返回当前工作目录的绝对路径

package.json是一个配置文件，它记录着项目的各种信息，如项目名称、项目依赖那些库等。

如何创建packa.json

package.json个字段含义

常见的属性

  • name：项目名称 必填
  • version：当前项目的版本号 必填
  • descrption：描述信息，很多时候是作为项目的基本描述
  • author：作者相关信息
  • license：开源协议
  • private：记录当前项目是私有的，当值为true时，npm不能发布他
  • main：设置程序入口 比如我们使用axios模块 const axios=require(‘axios’) 如果有main属性，实际上是找到对应的main属性文件

比较重要的字段

scripts属性：

  • scripts属性用于配置一些脚本命令，以键值对的形式存在，配置后我们可以通过npm run命令的key来执行这个命令
  • npm start 和npm run start的区别
    • 他们是等价的，对于常用的start,test,stop,restart可以shengl掉run直接通过npm start等方式运行

dependencies属性

dependencies属性是指无论开发环境还是生成环境都需要依赖的包

通常是我们项目实际开发用到的一些模块，与之对应的是devDependencies

直接使用npm install会安装dependencies里面的所有内容

devDependencies属性

一些包在生成环境（可以理解为浏览器所显示的页面）下是不需要的比如webpack，babel这个时候我们可以通过npm install webpack –save–devv(简写为npm install webpack -D)将他安装到devDependencies中

peerDependencies属性

还有一种项目依赖关系是对等依赖，也就是你依赖的一个包，它必须以另外一个包为前提的，比如element是基于vue3的

package-lock.json

文件解析

name:项目名称

version:项目版本

lockfileVersion：lock文件的版本

requires：使用requires来跟踪模块的依赖关系（true or false)

dependencies：项目的依赖

  • version：实际安装的版本
  • resolved：下载地址
  • requires/dependencies：记录当前模块的依赖
  • integrity：用来从缓存中换取索引，再通过索引去获取压缩包文件
