浏览器渲染原理

凭记忆写的，还是看专业的文章吧

在浏览器输入url后发生了什么

域名通过dns解析为ip地址，然后IP地址找到服务器，服务器返回一个html文件，在解析html文件过程中，如果碰到像link script这样的标签会再次进行下载

浏览器内核以及解析过程

浏览器内核是指浏览器的排版引擎 也被称为浏览器引擎，页面渲染引擎一个网页下载下来是由渲染引擎来解析的

解析过程

​

html解析生成domtree

script操作修改domtree

css解析生成各种css规则

两者结合生成rendertree，rendertree此时渲染树各个节点没有大小和摆放位置，需要layout过程完成（为了避免dom树上不显示的元素）

然后就可以展示在页面上了。

script和页面解析

在解析过程中，如果碰到script会停止解析，去下载js文件（css不会停止解析），下载完成后执行全部js代码，完成后继续解析。但是解析过程中用户什么都看不到。我们可以用defer 和 async属性解决或者将script放到底部或中间

  • defer：script代码下载和执行不会阻塞.domtree的构建，等domtree构建完成后，在DOMContentLoaded事件前先执行defer中的代码 如果有多个defer会保证script的执行顺序
  • async：完全独立，一旦下载完立刻执行，不会按照顺序

回流和重绘

回流：对节点大小，位置，改变dom结构等修改后需要再重新布局

重绘：对节点颜色改变等修改后不需要重新布局

如何避免回流

回流对浏览器性能影响太大，应当尽量避免

  • 修改样式时一次型修改（添加class cssText
  • 操作dom时一次性添加 （DocumentFragment
  • 使用绝对定位和相对定位 （还是会引起回流，但是影响比较小

面向对象三大特性：

对象的原型链

读取对象的某个属性时，JavaScript 引擎先寻找对象本身的属性，如果找不到，就到它的原型去找，如果还是找不到，就到原型的原型去找。如果直到最顶层的Object.prototype还是找不到，则返回undefined。如果对象自身和它的原型，都定义了一个同名属性，那么优先读取对象自身的属性，这叫做“覆盖”（overriding）这样一层一层向上查找就形成了原型链。

map数据结构

JavaScript 的对象（Object），本质上是键值对的集合（Hash 结构），但是传统上只能用字符串当作键。这给它的使用带来了很大的限制

代码原意是将一个 DOM 节点作为对象data的键，但是由于对象只接受字符串作为键名，所以element被自动转为字符串[object HTMLDivElement]

为了解决这个问题，ES6 提供了 Map 数据结构。它类似于对象，也是键值对的集合，但是“键”的范围不限于字符串，各种类型的值（包括对象）都可以当作键。也就是说，Object 结构提供了“字符串—值”的对应，Map 结构提供了“值—值”的对应，是一种更完善的 Hash 结构实现

注意，只有对同一个对象的引用，Map 结构才将其视为同一个键。这一点要非常小心。

代码的set和get方法，表面是针对同一个键，但实际上这是两个不同的数组实例，内存地址是不一样的，因此get方法无法读取该键，返回undefined

同理，同样的值的两个实例，在 Map 结构中被视为两个键

由上可知，Map 的键实际上是跟内存地址绑定的，只要内存地址不一样，就视为两个键

promise

Promise 是异步编程的一种解决方案，比传统的解决方案——回调函数和事件——更合理和更强大。它由社区最早提出和实现，ES6 将其写进了语言标准，统一了用法，原生提供了Promise对象

promise是一个类，可以翻译为承诺，许诺，期约；

当我们需要的时候，给与调用者一个承诺：待会我给你回调数据时，就可以创建一个promise对象；

再通过new创建Promise对象时，我们需要传入一个回调函数，我们称之为executor

  • 这个回调函数会被立即执行，并且给传入另外两个回调函数resolve，reject
  • 当我们调用resolve回调函数时，会执行Promise对象的then方法传入的回调函数时
  • 当我们调用reject回调函数时，会执行Promise对象的catch方法传入的回调函数时

promise各个状态区分

promise代码结构

上面promise的使用过程可以将它划分为三个状态

  • 待定（pending）：初始状态，即没有被兑现，也没有被拒绝；执行executor中代码时处于该状态
  • 已兑现（fulilled）：意味着操作成功完成；执行了resolve时处于该状态，promise已被兑现
  • 已拒绝（rejected）：意味着操失败；执行了reject时处于该状态，promise已被拒绝

resolve不同值的区别

  • 传入一个普通的值或对象，那么这个值会作为then回调的参数
  • 传入的是另外一个promise，那么这个新promise会决定原promise的状态
  • 如果resolve中传入的是一个对象，并且这个对象有实现then方法，那么会执行该then方法，并且根据then方法的结果来决定promise状态

finally方法：不接受参数，因为无论前面是fulfilled状态还是rejected状态，她都会执行

promise.then()

promise.then()本身返回一个promise，这个新的promise的决议是等到then方法传入的回调函数有返回值时，进行决议 ，如果遇到reject，会找最近的catch catch同理

await，async

异步函数

异步函数由async关键字定义，异步函数的内部代码执行过程和普通的函数是一致的，默认情况下也是会被同步执行

异步函数有返回值时，回合普通函数有区别

  • 异步函数也可以有返回值，但是异步函数的返回值相当于被包裹到Promise.resolve中
  • 如果我们的异步函数返回值是Promise，状态由会由Promise决定
  • 如果我们的异步函数的返回值是一个对象并且实现了thenable，那么由对象的then方法决定

如果在async中抛出了异常，会作为promise的reject进行传递

await关键字的使用

在异步函数中可以使用await关键字

通常使用await后面会接一个表达式，返回一个promise，然后等待promise有结果后才会继续执行后续代码 如果promise对象的结果是resolve，它会将resolve的值，作为await表达式的运算结果。
