版本控制

版本控制的英文是Version control，是维护工程蓝图的标准做法，能追下哦那个工程蓝图从诞生一直到定案的过程；版本控制也是一种软件工程技巧，借此能在软件开发的过程中，确保由不同人所编辑的同一程序文件都得到同步

版本控制的功能

  • 不同版本的存储管理
  • 重大版本的备份维护
  • 恢复之前的项目版本
  • 记录项目的点点滴滴
  • 多人开发的代码合并

GIT

Bash,CMD,GUI区别

Bash，Unix shell的一种，Linux与Mac OS X都将它作为默认shell

GitBash就是一个shell，是windows下的命令行工具，可以执行linux命令，他是基于CMD的，在CMD基础上增添一些新的命令和功能

Git CMD

命令行提示符（CMD）是windows操作系统上的命令行解释程序，当你在Windows上安装git并且习惯使用命令行时，可以使用cmd来运行git命令

Git GUI

他提供了一个图形用户界面来运行git命令

git本地仓库

初始化本地仓库git init

该命令将创建一个名为.git的子目录，这个子目录含有你初始化的Git仓库中所有的必须文件，这些文件是Git仓库的核心，但是这时候我们仅仅是做了一个初始化操作，你的项目文件还没有被跟踪

git clone +地址

该命令从其他服务器克隆一个已存在的Git仓库

文件状态划分

现在我们的电脑上已经有一个git仓库

  • 在实际开发中，你需要将某些文件由这个git仓库来管理

  • 并且我们之后会修改文件的内容，当达成某一个目标时，想要记录下来这次操作，就会将它提交到仓库中

那么我们需要对文件来划分不同的状态，以确定这个文件是否已经归于git仓库的管理

  • 未跟踪：默认情况下，Git仓库下的文件也没有添加到Git仓库管理中，我们需要通过add命令来操作 新建一个文件
  • 已跟踪：添加到Git仓库管理的文件处于已跟踪状态，Git可以对其进行各种跟踪管理

已跟踪的文件又可以进行细分状态划分

  • staged：暂缓区中的文件状态 git add命令 跟踪命令
  • Unmodified：执行commit命令，可以将staged中文件提交到Git仓库
  • Modified：修改了某个文件后，会处于Modified状态

检查文件状态

git status 会检查未跟踪的文件

git status -s可以显示简介信息

git忽略文件

一般我们总会有些自动生成的文件，比如日志文件，或者编译过程中创建的临时文件等；

我们可以创建一个名为.gitignore的文件 *他也需要被提交到仓库 *，列出要忽略的文件的模型

在实际开发中，这个文件不需要手动创建在必须的时候添加自己的忽略内容即可

git 校验和

Git中所有的数据在存储前都计算校验和，然后以校验和来引用

  • git 用以计算机校验和的机制叫做SHA-1散列；
  • 这是一个由40个十六进制字符组成的字符串，基于Git中文件的内容跟或目录结构计算出来

git log

可以使用git log查看所有历史提交记录，再不传入任何参数的默认情况下，git log会按时间先后顺序列出所有的提交，最近更新的排在最上面

可以使用 git log –pretty=oneline查看简洁信息

版本回退 -git reset

如果想要进行版本回退，我们需要先知道目前处于哪一个版本：git通过HEAD指针记录当前版本。

  • HEAD时当前分支引用的指针，他总是指向该分支上的最后一次提交；
  • 通过HEAD 的最简方式，就是将它看作该分支上的最后一次提交

我们可以通过HEAD来改变Git目前的版本指向：

  • 上一个版本就是 git reset –hard HEAD^
  • 如果是上1000个版本 git reset –hard HEAD~1000
  • 我们可以指定一个commit id（校验和）git reset –hard 2d4489 2

Git远程仓库 服务器

什么是远程仓库？

  • 之前我们的代码是保存在一个本地仓库中，也就意味着我们只是在进行本地操作
  • 在真实开发中，我们通常是多人开发的，所以我们会将管理的代码提交到远程仓库中
  • 远程仓库通常是搭建在一个服务器上的
  • 所以我们需要在git服务器上搭建一个远程仓库

目前我们有如下方式可以使用Git服务器

  • 使用第三方的Git服务器：GitHub，Gitee(码云)，GItlab
  • 在自己服务器搭建一个Git服务器

将远程仓库下载到本地

git clone+仓库地址

直接下载私有仓库会失败，因为我们没有凭证

远程仓库的验证

对于私有仓库我们想要进行操作，远程仓库会对我们的身份进行验证：

目前Git服务器验证手段主要由两种

  1. 基于HTTP的凭证存储
  2. 基于SSH的密钥

http的验证

因为本身http协议是无状态链接，所以每一个链接都需要用户名和密码

我们可以使用git credential mannager来登录

shh密钥验证

Secure Shell(安全外壳协议，简称SSH)是一种加密的网络传输协议，可在不安全的网络中为网络服务提供安全的传输环境.

SSH以非对称加密实现身份验证.

  • 例如其中一种方法是使用自动生成的公钥-私钥对来简单地加密网络连接，随后使用密码诀证进行登录;
  • 另一种方法是人工生成一对公钥和私钥，通过生成的密钥进行认证，这样就可以在不输入密码的情况下登录;
  • 公钥需要放在待访问的电脑之中，而对应的私钥需要由用户自行保管;

生成公钥密钥命令

在gitbash 里面输入命令

ssh-keygen -t ed25199 -C “your email”

之后会生成一个公钥在c盘

然后去远程仓库服务器添加公钥

管理远程服务器

查看远程仓库地址

  • git remote
  • git remote -v -v是-verbose的缩写（冗长的）

添加远程仓库地址（让远程仓库与本地仓库进行连接）

  • git remote add

    shortname是自己进行命名 url是仓库地址

然后使用git push 和 git pull

可能遇到的问题

  • 合并两个没有共同base（祖先）的分支
  • 本地分支没有跟踪上游分支

与远程仓库交互

  • git clone+远程仓库地址：将存储库克隆到新创建的项目中

  • 将代码push到远程仓库：将本地代码推送到远程仓库中

    默认情况下是将当前分支（master）push到origin远程仓库的

  • 从远程仓库中fetch代码：从远程仓库中获得最新代码

    默认情况下是从origin中获取代码

    获取代码后默认并没有合并到本地仓库，我们需要merge来合并
